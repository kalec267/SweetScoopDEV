Franchise Platform

SweetFlow System

지능형 자동 물류 인프라 및 실시간 게미피케이션 시스템 구축 제안서

아이스크림 전문점 프랜차이즈 효율화 플랫폼

Spring i18n 기반 다국어 처리, 루프형 세션 주문, 안전재고 임계치 자동 발주 및 데이터 시각화 분석 엔진이 통합된 무결성 백오피스 솔루션입니다.

01. 프로젝트 개요

Level 1 - Client

고객용 무인 키오스크

유연한 맛 선택 메커니즘을 반영한 직관적인 UI/UX. 프린터 하드웨어 통신 및 실시간 품절 처리(DB Lock 연동)가 구현된 POS 단말기입니다.

  • 미결제/결제완료 상태 분기
  • 60초 유휴 시간 시 자동 리셋
Level 2 - Branch

지점별 매장 운영 (POS)

지점장 전용 대시보드로 실시간 당일 매출 조회, 재고 추적, 본사 발주 승인 상태 확인이 가능합니다. 키오스크 영업 시작/중단 제어를 원격으로 처리합니다.

  • 실시간 재고 기반 품절 제어
  • 수동 영수증 출력 및 주문 취소
Level 3 - Headquarters

본사 총괄 백오피스

가맹점의 통합 주문 데이터 적재, 신규 지점 라이선스 발급 및 운영 제어, 본사 물류창고 입출고 정보 및 발주 승인 흐름을 관장하는 핵심 코어입니다.

  • 다차원 판매 실적 시각화
  • 발주 흐름 상태 제어 (5단계)

Standard Tech Stack & Integration

본 시스템은 비즈니스 안정성과 실시간 처리를 극대화하는 표준 기술로 구성됩니다.

Spring Boot / REST API MySQL (Transactional Lock) Firebase Cloud Messaging (FCM) Hardware SDK (Printer Integration) Vue.js (Framework)

03. 상세 요구사항 명세서

1. 소비자 단말 (Kiosk Client)

  • 스프링 연동 다국어: LocaleResolver 인터셉터 바인딩으로 텍스트 정보 실시간 i18n 메시지 매핑 전환.
  • 루프형 연속 주문 프로세스: 일회성 담기 방지. 컵 선택 후 맛을 담고 장바구니에 적재한 뒤, 세션 단절 없이 다시 홈으로 회귀하여 콘 선택 흐름을 유연하게 계속 수행 가능.
  • 유휴 세션 리셋 제어: 60초간 화면 입력 조작 공백 발생 시 장바구니 리셋 및 초기 메인 스크린으로 강제 복귀.
  • 4단계 주문 라이프 트래킹: 결제 완료 건은 접수 → 대기중 → 픽업대기 → 픽업완료 플래그를 타며 FCM으로 화면에 실시간 UI 동기화 표출.
  • 한도 한정 룰렛 리워드: 결제 종료 후 보상 이벤트 창 가동. 본사 설정 일일 최대 당첨수 검증 로직 반영 및 초과 시 꽝 세그먼트 강제 분기.

2. 분점 관리자 (Branch POS)

  • 실시간 수납 및 알림 처리: 현금(unpaid 가주문 상태) 및 카드(paid 즉시완료) 데이터 FCM 즉시 수신 및 원장 등록.
  • 임계치 기반 자동 지능 발주: 특정 맛 벌크 재고가 설정된 하한 임계치(safety_threshold) 이하로 감소 시, 시스템 내부 트랜잭션이 작동하여 본사 시스템에 PENDING 상태의 stock_requests를 자동으로 Insert 처리함.
  • 주문 취소 및 환불 보상: 변심 환불 시 단일 작업 내에서 대금 환원 및 차감됐던 벌크 재고 원상 복구 트랜잭션 보장.
  • 본점 소통 및 직원 호출: 1:1 문의 채널 및 키오스크 연동 직원호출 푸시 접수 인터페이스 제공.

3. 총관리자 및 시스템 동기화

  • 원격 마케팅 배포 제어: 중앙에서 이벤트 프로모션 비주얼 배너 이미지, 이달의 맛 홍보 원장, 시간대별 타임 세일 할인율을 구성 설정하여 전국 키오스크 메뉴판 가용성 및 디스플레이 노출 순위를 실시간 일괄 업데이트함.
  • 발주 상태 흐름 관리: 가맹 지점의 수동/자동 신청 내역 필터링 및 본사 물류창고 자체 재고 신청 보완.
  • 데이터베이스 동기화 기조: 본사 물류 센터 현황과 분점의 발주 상태가 전이되는 물리 시점에 REST API 비동기 통신 모델을 연동하여 분산 환경 무결성을 획득할 예정.

05. 기능별 상세 세부 명세서

Notion Spec-Sheet Database
U-001 ~ U-003 / 선형 루프 주문 및 포인트

매장/포장 수령 선택 후 컵/콘 사이즈를 지정합니다. 컵 구성을 마쳐도 세션이 단절되지 않고 메인으로 복귀하여 다른 콘 상품을 연속 적재할 수 있는 루프형 카트 엔트리를 구축하고 결제 전 멤버십 번호를 수집합니다.

U-004 ~ U-005 / 다국어 지원 및 하드웨어 트랜잭션

스프링 i18n 로케일 해석 파라미터(?lang=) 연동으로 메뉴판과 안내 패널 텍스트를 무부하 실시간 대체 결합합니다. 대금 지급 승인 시 영수증 프린터 호출, 재고 차감 처리를 하나의 트랜잭션으로 보장해 에러 발생 시 롤백합니다.

U-006 ~ U-007 / 실시간 4단계 관제 및 룰렛 검증

결제가 완료된 주문은 주방 디스플레이(KDS) 및 고객 스크린에 [접수 → 대기중 → 픽업대기 → 픽업완료]의 흐름으로 표출됩니다. 결제 직후 룰렛 창이 연동되며 일일 당첨 상한 도달 검증 규칙에 의거해 감면 리워드를 발행합니다.

U-008 / 키오스크 유유 타임아웃 및 호출

지정 조작 유휴 시각(60초) 도달 시 다음 사용자의 정보 보호를 위해 장바구니 스토리지를 완전 초기화 퍼지 조치합니다. 긴급 장애 조치 헬프를 위한 '직원 호출' 버튼 클릭 시 대리점 포스로 신호를 타격합니다.

AC-001 / 보안 및 세션 정책

지점장과 최고 권한자는 단일 접근 엔트리를 경유하며 ID/PW 비교 인증 단계를 거칩니다. 세션 만료 시간 정책이 적용됩니다.

AC-002 / 다차원 복합 인덱스 조회

대용량 판매 이력 조회 지연을 차단하고자 데이터베이스 생성일 및 매장 복합 식별자 인덱싱 최적화 튜닝을 수행합니다.

AC-003 / 1:1 업무 소통 게시판

본사 대 대리점 간 업무 요청 질의 및 가맹 피드백을 실시간 기록 보존하며, 등급별 로우 레벨 필터링 수정 제한 조건을 적용합니다.

AS-001 ~ AS-003 / 매장 운영 제어 및 환불 트랜잭션

아이디/비번 로그인 후 영업 시작을 고지하며 당일 실시간 매출/재고를 트랙킹합니다. 예외 시 특정 품목을 수동 품절 처리하여 키오스크 렌더러에 즉시 반영하고, 변심 취소 시 재고 가산 트랜잭션 무결성을 수행합니다.

AS-004 ~ AS-006 / 자율 지능형 자동 발주 시스템 및 인쇄

기존 로그 원장을 기준으로 안전 영수증을 수동 재출력합니다. 매장 벌크 수량이 사전에 설정된 최저 하한선 이하 점포 임계값에 도달 시, 백그라운드 스케줄러가 자동으로 본사에 PENDING 상태의 stock_requests 내역 데이터를 자동 생성해 등록합니다.

AH-001 ~ AH-003 / 대리점 거버넌스 및 발주 흐름 제어

전국 체인점의 추가/수정 및 가동 등급 상태([정상/대기/중단])를 실시간 제어하며 분점의 자율 자동 상신 발주 문건을 모니터링하여 승인/반려/배송 단계 흐름 상태값 변화 프로세스를 통제하고 긴급 유선 누락분 데이터를 총관리자가 직접 대리 기입 수동 등록합니다.

AH-004 ~ AH-006 / 본사 중앙 물류 관리 및 마케팅 원격 배포

중앙 공장 제조 완료 입고분을 원장에 동기화하고 물류량을 업데이트합니다. 본사 마케팅 엔진 단에서 타깃 프로모션 이미지 배너 파일(.png) 배포 규칙 및 비인기 오후 시간대 자동 할인 차감율, 신메뉴 이달의 맛 노출 우선순위 스케줄러를 등록 조율하면 전체 가맹점 메뉴판 인프라에 즉시 동기화 업데이트 적용됩니다.

05. 홍보 통계 데이터 명세 및 기대 효과

01. 이달의 맛 홍보 데이터

속성: 이달의 맛 ID, 이름, 맛 설명, 대표 이미지(.png), 판매 시작/종료일, 메뉴판 노출 순서, 판매 여부

활용 방안: 새로 추가된 이달의 맛을 메뉴판 첫 번째에 고정 노출하여 신메뉴 홍보 및 구매를 적극적으로 유도합니다.

02. 인기 맛 통계 데이터

속성: 맛 ID, 이름, 누적 판매량, 판매 순위, 최근 한 달 판매량, 인기 메뉴 TOP1~TOP5 정보

활용 방안: 판매 데이터를 기반으로 인기 맛 순위를 산정하고, 메뉴판 옆에 TOP1~TOP5를 표시하여 고객의 메뉴 선택을 돕고 구매를 유도합니다.

03. 구매 시간대 통계 데이터

속성: 주문 ID, 주문 일시, 주문 시간대, 시간대별 주문 건수, 시간대별 매출액, 평균 주문 금액

활용 방안: 시간대별 구매 데이터를 분석하여 주문량이 적은 시간대를 파악하고, 해당 시간대에 할인 이벤트를 진행하여 매출을 증대시킵니다.

04. 할인 이벤트 데이터

속성: 이벤트 ID,명칭, 할인 적용 시작/종료 시간, 할인율, 적용 대상 메뉴, 이벤트 활성화 여부

활용 방안: 구매율이 낮은 시간대에 자동으로 할인 이벤트를 적용하여 유휴 타임슬롯의 오프라인 고객 방문을 능동적으로 유도합니다.

05. 룰렛 이벤트 데이터

속성: 이벤트 ID, 명칭, 룰렛 보상 종류, 당첨 확률, 당첨 인원 제한, 하루 당첨 가능 인원, 시작/종료일, 활성 여부

활용 방안: 주문 완료 후 룰렛 이벤트를 제공하여 재방문을 촉진합니다. 하루 당첨 인원을 엄격히 제한하여 이벤트 희소성을 높이고, 보상은 할인 쿠폰, 토핑 무료, 사이즈 업, 1+1 쿠폰 등으로 구성하여 홍보 효과를 강화합니다.

-- ① 지점 및 권한 관련 테이블
CREATE TABLE branches (
    id INT AUTO_INCREMENT COMMENT '분점 고유 번호',
    name VARCHAR(100) NOT NULL COMMENT '분점명 (예: 강남역점)',
    status VARCHAR(20) NOT NULL DEFAULT 'OPERATING' COMMENT '운영 상태 (OPERATING / BREAK / CLOSED)',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '등록일',
    PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='분점 정보';

CREATE TABLE managers (
    id INT AUTO_INCREMENT COMMENT '관리자 고유 번호',
    branch_id INT NULL COMMENT '소속 분점 ID (본사 총관리자는 NULL)',
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '로그인 ID',
    password VARCHAR(255) NOT NULL COMMENT '암호화된 비밀번호',
    role VARCHAR(30) NOT NULL COMMENT '권한 (SUPER_ADMIN / BRANCH_MANAGER)',
    PRIMARY KEY (id),
    CONSTRAINT fk_managers_branch_id FOREIGN KEY (branch_id) REFERENCES branches (id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='관리자 계정';

-- ② 상품 및 메뉴 관련 테이블
CREATE TABLE product (
    id INT AUTO_INCREMENT COMMENT '상품 고유 번호',
    name VARCHAR(100) NOT NULL COMMENT '상품/맛 이름 (예: 엄마는 외계인, 아메리카노)',
    category VARCHAR(30) NOT NULL COMMENT '상품의 종류 (ICECREAM, COFFEE, BEVERAGE, DESSERT)',
    is_available BOOLEAN NOT NULL DEFAULT TRUE COMMENT '판매 가능 여부',
    PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='상품 및 아이스크림 맛 정보';

CREATE TABLE sizes (
    id INT AUTO_INCREMENT COMMENT '사이즈 고유 번호',
    name VARCHAR(50) NOT NULL COMMENT '사이즈명 (예: 싱글레귤러, 파인트, 쿼터)',
    flavor_count INT NOT NULL DEFAULT 0 COMMENT '선택 가능한 맛 개수 (완제품은 0)',
    price DECIMAL(10, 0) NOT NULL COMMENT '사이즈별 기본 가격',
    PRIMARY KEY (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='사이즈 및 기본 가격';

-- ③ 주문 및 결제 관련 테이블
CREATE TABLE kiosks (
    id INT AUTO_INCREMENT COMMENT '키오스크 고유 번호',
    branch_id INT NOT NULL COMMENT '소속 분점 ID',
    device_name VARCHAR(50) NOT NULL COMMENT '기기 명칭 (예: 정문 앞 1호기)',
    is_active BOOLEAN NOT NULL DEFAULT TRUE COMMENT '활성화 여부',
    PRIMARY KEY (id),
    CONSTRAINT fk_kiosks_branch_id FOREIGN KEY (branch_id) REFERENCES branches (id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='키오스크 기기 관리';

CREATE TABLE orders (
    id INT AUTO_INCREMENT COMMENT '주문 고유 번호',
    branch_id INT NOT NULL COMMENT '주문이 발생한 분점 ID',
    kiosk_id INT NULL COMMENT '주문이 발생한 키오스크 ID',
    order_number VARCHAR(50) NOT NULL COMMENT '영수증 출력용 주문 번호',
    pickup_type VARCHAR(20) NOT NULL COMMENT '수령 방식 (STORE / PACKING)',
    payment_type VARCHAR(20) NOT NULL COMMENT '결제 방식 (CASH / CARD)',
    payment_status VARCHAR(20) NOT NULL DEFAULT 'UNPAID' COMMENT '결제 상태 (UNPAID / PAID)',
    total_price DECIMAL(10, 0) NOT NULL DEFAULT 0 COMMENT '총 결제 금액',
    customer_phone VARCHAR(20) NULL COMMENT '포인트 적립용 전화번호',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '주문 일시',
    PRIMARY KEY (id),
    CONSTRAINT fk_orders_branch_id FOREIGN KEY (branch_id) REFERENCES branches (id),
    CONSTRAINT fk_orders_kiosk_id FOREIGN KEY (kiosk_id) REFERENCES kiosks (id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='주문 메인 정보';

CREATE TABLE order_items (
    id INT AUTO_INCREMENT COMMENT '주문 상세 고유 번호',
    order_id INT NOT NULL COMMENT '연결된 주문 메인 ID',
    size_id INT NULL COMMENT '선택한 사이즈 ID',
    container_type VARCHAR(20) NOT NULL COMMENT '용기 타입 (CUP / CONE / WAFFLE_CONE / NONE)',
    price DECIMAL(10, 0) NOT NULL COMMENT '구매 당시 1개당 단가',
    quantity INT NOT NULL DEFAULT 1 COMMENT '해당 구성의 주문 수량',
    PRIMARY KEY (id),
    CONSTRAINT fk_order_items_order_id FOREIGN KEY (order_id) REFERENCES orders (id) ON DELETE CASCADE,
    CONSTRAINT fk_order_items_size_id FOREIGN KEY (size_id) REFERENCES sizes (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='주문 상세(통 단위)';

CREATE TABLE order_item_flavors (
    id INT AUTO_INCREMENT COMMENT '고유 번호',
    order_item_id INT NOT NULL COMMENT '주문 상세 ID',
    icecream_id INT NOT NULL COMMENT '선택한 맛 ID (product.id 매핑)',
    PRIMARY KEY (id),
    CONSTRAINT fk_flavors_order_item_id FOREIGN KEY (order_item_id) REFERENCES order_items (id) ON DELETE CASCADE,
    CONSTRAINT fk_flavors_icecream_id FOREIGN KEY (icecream_id) REFERENCES product (id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='주문별 아이스크림 맛 매핑';

CREATE TABLE payments (
    id INT AUTO_INCREMENT COMMENT '결제 고유 번호',
    order_id INT NOT NULL COMMENT '주문 ID',
    payment_method VARCHAR(20) NOT NULL COMMENT '결제 수단',
    amount DECIMAL(10, 0) NOT NULL COMMENT '해당 수단으로 결제된 금액',
    transaction_number VARCHAR(100) NULL COMMENT '카드 승인 번호 또는 바코드 번호',
    status VARCHAR(20) NOT NULL DEFAULT 'APPROVED' COMMENT '결제 상태 (APPROVED / CANCELLED)',
    paid_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '결제 일시',
    PRIMARY KEY (id),
    CONSTRAINT fk_payments_order_id FOREIGN KEY (order_id) REFERENCES orders (id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='결제 상세 이력';

-- ④ 쿠폰 및 포인트 관련 테이블
CREATE TABLE coupons (
    id INT AUTO_INCREMENT COMMENT '쿠폰 고유 번호',
    code VARCHAR(50) NOT NULL UNIQUE COMMENT '쿠폰 바코드 번호',
    name VARCHAR(100) NOT NULL COMMENT '쿠폰 이름',
    discount_type VARCHAR(20) NOT NULL COMMENT '할인 유형',

CREATE TABLE point_histories (

    CONSTRAINT fk_point_histories_order_id FOREIGN KEY (order_id) REFERENCES orders (id) ON DELETE SET NULL

CREATE TABLE used_coupons (
    id INT AUTO_INCREMENT COMMENT '사용 이력 고유 번호',
    order_id INT NOT NULL COMMENT '사용된 주문 ID',
    coupon_id INT NOT NULL COMMENT '사용한 쿠폰 ID',
    used_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '사용 일시',
