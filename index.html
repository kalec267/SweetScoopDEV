아이스크림 전문점 프랜차이즈 운영 효율화

실시간 재고 관리, 가맹점 발주 제어, 그리고 키오스크 디바이스 제어를 완벽하게 통합하는 차세대 통합 백오피스 및 클라이언트 아키텍처 제안서입니다.

01. 프로젝트 개요

Level 1 - Client

고객용 무인 키오스크

유연한 맛 선택 메커니즘을 반영한 직관적인 UI/UX. 프린터 하드웨어 통신 및 실시간 품절 처리(DB Lock 연동)가 구현된 POS 단말기입니다.

  • 미결제/결제완료 상태 분기
  • 60초 유휴 시간 시 자동 리셋
Level 2 - Branch

지점별 매장 운영 (POS)

지점장 전용 대시보드로 실시간 당일 매출 조회, 재고 추적, 본사 발주 승인 상태 확인이 가능합니다. 키오스크 영업 시작/중단 제어를 원격으로 처리합니다.

  • 실시간 재고 기반 품절 제어
  • 수동 영수증 출력 및 주문 취소
Level 3 - Headquarters

본사 총괄 백오피스

가맹점의 통합 주문 데이터 적재, 신규 지점 라이선스 발급 및 운영 제어, 본사 물류창고 입출고 정보 및 발주 승인 흐름을 관장하는 핵심 코어입니다.

  • 다차원 판매 실적 시각화
  • 발주 흐름 상태 제어 (5단계)

Standard Tech Stack & Integration

본 시스템은 비즈니스 안정성과 실시간 처리를 극대화하는 표준 기술로 구성됩니다.

03. 핵심 요구사항 명세서

비기능 및 시스템 예외 처리 정책 포함

1. 키오스크 & 결제 프로세스

  • 선택 방식의 계층화 수령(매장/포장) → 컵/콘/사이즈 지정 → 규격에 따른 실시간 맛 매핑 선택.
  • 유휴 세션 리셋 (Kiosk Timeout) 60초간 응답이 없을 경우, 개인정보 보호 및 다음 대기 승객을 위해 장바구니 자동 비우기 및 시작 화면 복귀.
  • 결제 수단 분기 & 상태 현금 주문 시 '미결제(UNPAID)'로 기록하여 직원이 카운터 수납 후 POS에서 승인. 카드/모바일은 즉시 완료(PAID).

2. 하드웨어 연동 및 데이터 정합성

  • 원자적(Atomicity) 트랜잭션 관리 결제 승인, 실시간 재고 차감, 영수증 프린터 전송 처리를 단일 비즈니스 트랜잭션으로 래핑. 물리 오류 시 전량 자동 롤백.
  • 하드웨어 장애 모니터링 용지 부족(Paper-out), 프린터 커버 오픈, 네트워크 타임아웃 감지 즉시 운영 대시보드 및 키오스크 화면에 에러 레이어 표출.
  • 실시간 푸시 알림 FCM 및 Firebase Realtime 연동으로 주방 디스플레이(KDS)와 점포 POS 단말기에 승인 주문 즉각 렌더링.

3. 재고 동시성 및 물류 동기화

  • 비관적 락(Pervasive Lock) 도입 인기 품목 마지막 한 통에 다수의 주문이 집중될 시, DB 수준에서 SELECT ... FOR UPDATE 구문을 적용하여 초과 판매(Over-selling) 원천 차단.
  • 실시간 재고 연동 흐름 재고 수량이 0이 되는 즉시, 키오스크 UI 내에 해당 품목에 대한 '품절(Sold-out) 처리' 스크립트 트리거.
  • 본사 물류 및 가맹점 발주 동기화 각 분점의 발주 신청에 따른 상태 변화(신청 → 접수 → 배송준비 → 배송중 → 완료) 시 본사 REST API와 실시간 동기화하여 투명한 적재 상태 모니터링 제공.

04. 사용자별 기능 정의서

ID 대상 기능명 기능 정의 및 상세 설명 비즈니스 로직 및 예외 처리 정책
U-001 소비자 메뉴 조회 지점 키오스크에서 가용한 판매 상품/사이즈 정보를 화면에 출력 지점 재고 정보 중 quantity > 0인 항목만 화면에 동적으로 렌더링하고, 품절 상태를 실시간 연동.
U-002 소비자 장바구니 결제 장바구니에 적재된 메뉴 리스트 정보를 바탕으로 주문 데이터 생성 및 대금 지급 처리 트랜잭션 시작 후 SELECT ... FOR UPDATE를 통해 다중 결제 시점에 재고 소진 시 롤백 및 '선택 맛 변경' 팝업 발생.
U-003 소비자 포인트 적립 전화번호 기반 회원 데이터베이스 식별 후 결제액에 의거한 멤버십 포인트(10%) 적립 휴대폰 번호 형식 유효성 체크(Regex) 및 최초 로그인 시 자동 가입 유입 후 이력 관리 테이블에 즉시 기록.
U-004 소비자 영수증 출력 결제 승인 직후 USB/LAN 프로토콜 통신을 통해 주방 및 고객 영수증 프린터 디바이스 가동 프린터 상태 에러 코드 감지 시 DB 주문 승인은 정상 처리하되, '미출력 주문건' 리스트로 적재 및 POS에 즉시 경고 알림 전송.
AC-001 관리자 공통 로그인 인가된 사용자가 자격증명 정보(ID/PW)를 기반으로 시스템에 접근하고 인증 처리 완료 보안을 위해 비밀번호 해시 일치 여부를 검증하고 토큰 발급. 세션 및 토큰 만료 시간(Session Timeout) 정책을 엄격히 적용 및 세션 탈취 예방.
AC-002 관리자 공통 통계 조회 지점 및 본사 단위에서 각자 열람 권한이 승인된 매출, 판매 이력 및 흐름 리포트를 조회 대규모 데이터 분석 시 데이터 레이턴시 예방을 위해 통계 쿼리 실행 계획 검증 및 인덱싱(생성일, 지점 ID 복합 인덱스 등) 조회 최적화.
AS-001 지점장 키오스크 활성화 매장 영업 준비 완료 후 관리 계정을 통해 특정 키오스크 기기를 활성화 상태로 전환 지점 테이블의 status = 'OPERATING' 체크, 키오스크 기기 테이블 상태값 동시 업데이트 및 핑 통신 점검.
AS-002 지점장 실시간 재고 관리 지점 아이스크림 벌크 잔여량을 확인하고 필요 시 특정 품목을 수동 품절 조치 수동 품절 즉시 키오스크에 WebSocket/SSE 알림을 전송하여 고객 주문 화면을 실시간으로 새로고침 처리함.
AS-003 지점장 주문 취소/환불 결제 완료 건에 대한 환불 승인 프로세스를 진행하여 카드 취소 및 판매 이력을 정정 결제 수단 환불 트랜잭션 수행과 함께 주문 아이템에 해당하는 맛 수량을 지점 재고 테이블(branch_stocks)에 즉각 원복 처리.
AS-004 지점장 영수증 재출력 프린터 용지 걸림 등으로 영수증 소실 시 POS 기기에서 완료된 주문을 조회해 수동으로 영수증 재출력 해당 주문번호 정보의 상세 데이터를 조회한 후 프린터 하드웨어로 원격 출력 명령 재전송 (중복 결제 방지 처리).
AS-005 지점장 발주 신청 부족 재고 확보를 위해 본사로 맛 종류별 필요 수량을 지정해 재고 배송 신청 전송 PENDING 상태로 발주 신청 기록 생성. 본사의 접수 및 배송 중 처리에 따라 상태 변경 사항을 실시간 확인 가능.
AH-001 본사 분점 관리 신규 지점 추가, 지점 정보 수정 및 지점 운영상태 원격 제어 특정 가맹점 운영정지 처리 시 해당 가맹점 키오스크의 활성화 세션을 즉시 강제 로그아웃 조치하는 API 동시 연동.
AH-002 본사 발주 통합 관리 각 지점에서 전송한 발주 건들의 승인/반려 여부를 판단하고 물류 상태를 5단계로 통제 발주 승인 시 본사 물류창고 재고 수량을 차감하고 배송 프로세스로 변경 처리. 재고 부족 시 자동 반려 통보.
AH-003 본사 수동 발주 등록 전화 및 긴급 유선 요청에 따른 발주 신청 건에 대해 총관리자가 직접 데이터베이스에 수동 기록 본사 권한으로 등록되므로 이력 추적 필드(requested_by = manager_id)에 총관리자 ID를 필수 기록하여 무결성 보장.
AH-004 본사 본사 재고 관리 본사 중앙 물류센터의 입고 예정 정보 반영 및 현재 자산 가치가 반영된 전체 아이스크림 품목 보유량 통제 입고 시 물류창고 이력 기록 테이블과 대조하여 자동 합산 처리 및 대량 유통 기한 등 정밀 속성 확장 가능 구조로 설계.

05. 기능별 상세 세부 명세서

Notion Spec-Sheet Database
U-001 / 메뉴 및 맛 선택

수령 방식(매장/포장) 선택 후 컵/콘 및 용기 지정. 지점별 실시간 잔여 통수 기반 데이터 매핑을 통해 품절 제품은 카테고리에서 비활성화 레이어로 자동 전환처리.

U-002 / 비관적 락 트랜잭션

여러 키오스크 단말에서 동일한 마지막 1개 벌크 아이스크림을 선택하고 결제 요청 시, 동시성 오버셀링을 완화하기 위해 Spring Boot 단에서 데이터베이스 베타 락을 확보해 트랜잭션 보장.

U-003 / 전화번호 멤버십 연동

최종 결제 완료 직전, 10키패드 가상 키보드를 제공하여 전화번호 입력 후 가입 여부 체크. 미가입 전화번호인 경우 회원 마스터에 자동 레코드 추가 및 결제액의 10% 포인트 적립 처리.

U-004 / 프린터 연동 에러 핸들링

결제 도중 영수증 프린터의 용지 소진 혹은 통신 이탈 시, 전체 거래가 취소되는 장애를 방지하기 위해 '결제 성공' 로그는 보존하되, 알림 유도 및 지점 포스기 미출력 보관함에 보존.

AC-001 / 계정 보안 및 세션 제어

지점장과 본사 최고관리자는 동일한 백오피스 엔트리 포인트를 공유하며, JWT 토큰 방식을 통해 권한 인가 분기. 유휴 조작 시간 감지 시 자동으로 세션을 파기하고 로그인 유도.

AC-002 / 매출 데이터 다차원 시각화

일자별/주간별/월간별 판매량 집계 쿼리를 캐싱 처리하고 통계 뷰어 로딩 부하를 단축. 대량의 주문 트랜잭션을 일자 단위로 요약해 보관하는 통계 전용 파티셔닝 구조 설계.

AS-001 / 키오스크 전격 영업 통제

점포 준비 후 점포 POS 대시보드에서 원격으로 키오스크의 활성화 시그널 전송. 디바이스의 하트비트 체크 API를 가동하여 매장의 오프라인 상태 긴급 복구 메커니즘을 지원.

AS-002 / 수동 강제 품절 조치

현장 아이스크림 기기 파손 혹은 변질 등의 긴급 예외 발생 시, 지점장 POS에서 터치 한 번으로 즉시 품절 조치를 가동하며, 키오스크 서버에서 SSE 이벤트를 수신하여 화면을 실시간 갱신.

AS-003 / 보상 트랜잭션 주문 취소

고객 변심 및 키오스크 주문 착오 시 주문을 취소하면, 차감되었던 벌크 아이스크림 재고가 branch_stocks 테이블의 복합 키에 정확히 반사 및 자동 덧셈 연산 처리되어 정합성을 유지.

AS-004 & AS-005 / 발주 및 영수증 재출력

실시간 재고 경고치 이하로 잔량이 떨어지는 경우 지점장에게 발주 유도 모달 띄우기 및 발주 기록을 PENDING 상태로 전송. 출력 장애 대응을 위한 주문번호 기준 영수증 재발급 루틴 보장.

AH-001 / 지점 수명 주기 제어

전국 체인점의 라이선스, 운영자 계정, 해당 점포의 운영 상태 플래그(OPERATING / BREAK / CLOSED)를 원격 모의 처리하며, 비인가 점포 단말의 결제 차단용 글로벌 필터 인프라 가동.

AH-002 / 발주 처리 5단계 흐름 제어

각 지점의 발주 요청 건을 PENDING에서 REJECTED 또는 PREPARING으로 변경. 승인 완료 시 본사 창고 재고인 headquarters_stocks에서 대량 차감 후 지점에 발송 상태값 동기화 알림 발송.

AH-003 / 오프라인 수동 발주 구제

시스템 사용이 불가능하거나 오프라인 수동 긴급 요청의 경우, 총괄 관리자 권한을 이용하여 강제 레코드를 추가하고 생성 이력 필드(requested_by)에 본사 원장 ID를 필수로 로깅해 무결성을 담보.

AH-004 / 중앙 물류 재고 모니터링

원자재 수입 및 제조 공장을 통한 중앙 물류 수량 확보 시 대량 입고 배치 쿼리를 가동하여, 전사 자산 실사 및 가맹점에 가용한 가상 생산 가능량의 마진율을 즉각 계산함.

    `total_price` DECIMAL(10, 2) NOT NULL COMMENT '최종 총 주문 결제 금액',

CHAPTER 02

실시간 및 데이터 안정성 솔루션

고품질의 서비스 유지를 위한 비기능적 하드웨어 및 데이터베이스 무결성 확보 정책입니다.

SELECT ... FOR UPDATE (DB Lock)

다수의 고객이 마지막 잔여 1개의 맛 제품을 동시 결제할 시 발생할 수 있는 오버셀링을 막기 위해 트랜잭션 수준에서 비관적 락을 적용하여 트랜잭션 일관성을 고수합니다.

원자적 롤백 & 하드웨어 핸들링

PG사 결제가 되었더라도 프린터 하드웨어 통신 에러 등 연동 도중 에러 발생 시 데이터가 꼬이는 현상을 미연에 예방하고자 전체 트랜잭션을 일괄 원자적 롤백하거나 보상 트랜잭션을 적용합니다.

CHAPTER 03

상세 기능 명세 요약(U/AC/AS/AH)

각 비즈니스 수명주기별로 최적화된 프로세스 예외 핸들링 구조를 슬라이드 형태로 요약 제공합니다.

소비자(U-Series) 실시간 재고 연동 메뉴조회 및 비관적 락 동시성 제어 결제, 가상 키패드 기반 멤버십 적립, 프린터 하드웨어 통신 예외 트랜잭션 분리.
관리자 공통(AC-Series) 지점장과 본사 관리자의 JWT 보안 토큰 권한 인가 분기 및 세션 만료 시간 정책 수립. 대용량 파티셔닝 기반 인덱스 고도화 및 캐시 통계.
지점장(AS-Series) 점포 POS를 통한 키오스크 활성화 핑 제어, 긴급 품절 SSE 단방향 전파, 변심 취소 시 재고 자동 가산 트랜잭션 보장.
본사(AH-Series) 전국 지점의 정상/대기/중단 라이선스 수명 주기 제어, 지점 발주의 5단계 물류 트랙킹 및 수동 복구 강제 로깅.
CHAPTER 04

지점 및 주문 관계 설계 (3-Tier Layer)

하나의 주문 아래 다수의 용기 아이템(order_items)이 포함되고, 해당 용기 내에 여러 맛 상품(order_item_flavors)이 매핑되는 정형 설계를 반영했습니다.

물리 테이블명 필드 관계 상세
orders 고객 수령방식, 결제구분(CASH, CARD), 폰번호 수록
order_items 구매 사이즈(파인트/쿼터 등), 컵/콘/와플 등 하위 용기 매핑
order_item_flavors 사용자가 선택한 N개 아이스크림 맛(product) 매핑 기록
