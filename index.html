Franchise Platform

SweetScoop System

지능형 자동 물류 인프라 및 실시간 게미피케이션 시스템 구축 제안서

아이스크림 전문점 프랜차이즈 효율화 플랫폼

Spring i18n 기반 다국어 처리, 루프형 세션 주문, 안전재고 임계치 자동 발주 및 데이터 시각화 분석 엔진이 통합된 무결성 백오피스 솔루션입니다.

01. 프로젝트 개요

Level 1 - Client

고객용 무인 키오스크

유연한 맛 선택 메커니즘을 반영한 직관적인 UI/UX. 프린터 하드웨어 통신 및 실시간 품절 처리(DB Lock 연동)가 구현된 POS 단말기입니다.

  • 미결제/결제완료 상태 분기
  • 60초 유휴 시간 시 자동 리셋
Level 2 - Branch

지점별 매장 운영 (POS)

지점장 전용 대시보드로 실시간 당일 매출 조회, 재고 추적, 본사 발주 승인 상태 확인이 가능합니다. 키오스크 영업 시작/중단 제어를 원격으로 처리합니다.

  • 실시간 재고 기반 품절 제어
  • 수동 영수증 출력 및 주문 취소
Level 3 - Headquarters

본사 총괄 백오피스

가맹점의 통합 주문 데이터 적재, 신규 지점 라이선스 발급 및 운영 제어, 본사 물류창고 입출고 정보 및 발주 승인 흐름을 관장하는 핵심 코어입니다.

  • 다차원 판매 실적 시각화
  • 발주 흐름 상태 제어 (5단계)

Standard Tech Stack & Integration

본 시스템은 비즈니스 안정성과 실시간 처리를 극대화하는 표준 기술로 구성됩니다.

Spring Boot / REST API MySQL (Transactional Lock) Firebase Cloud Messaging (FCM) Hardware SDK (Printer Integration) Vue.js (Framework)

03. 상세 요구사항 명세서

1. 소비자 단말 (Kiosk Client)

  • 다단계 주문 결제 프로세스: 사용자는 직관적인 UI를 통해 메뉴 선택 → 맛 선택 → 컵/콘 및 사이즈 선택 흐름으로 주문을 구성할 수 있어야 한다.
  • 복합 결제 수단 및 영수증 연동: 현금 및 카드 결제 방식을 선택할 수 있으며, 결제 완료 시 하드웨어 연동을 통한 영수증 자동 출력이 지원되어야 한다.
  • 휴대폰 번호 연동 포인트 적립: 결제 시 휴대폰 번호 입력을 통한 포인트 적립 기능이 제공되며, 회원/비회원 분기 처리를 통해 멤버십 데이터 무결성을 유지한다.
  • 루프형 장바구니 및 유휴 리셋: 세션 단절 없이 다수 상품을 적재 가능하며, 60초 유휴 시 장바구니를 안전하게 강제 초기화(Clear) 해야 한다.

2. 분점 관리자 (Branch POS)

  • 키오스크 영업 상태 중앙 제어: 포스기에서 키오스크의 영업 시작 및 종료 상태를 원격으로 제어하여 단말기 활성화 여부를 결정한다.
  • 주문 관리 및 결제 취소: 고객 주문 접수 확인 및 상태 업데이트가 실시간으로 이루어지며, 예외 상황 시 주문 취소 및 환불 트랜잭션을 원자적으로 처리한다.
  • 다차원 매출 통계 조회: 일일 매출, 월간 누적 매출 및 개별 메뉴별 판매량 통계 지표를 시각화하여 제공한다.
  • 로컬 재고 및 발주 관리: 매장 내 입고/폐기 등록을 통한 실재고를 관리하고, 본사 시스템에 발주 신청을 트리거하는 물류 제어 인터페이스를 지원한다.

3. 본사 총괄 관리자 (HQ Admin)

  • 가맹점 및 마스터 데이터 거버넌스: 신규 지점 등록/수정/삭제와 전사적 메뉴 및 카테고리(CRUD) 정보를 중앙에서 통제하고 하위 레이어로 즉시 배포한다.
  • 물류 승인 및 본사 재고 인프라: 전국 분점의 발주 요청 건을 검토하여 승인/반려 상태를 제어하고, 중앙 물류 창고의 입출고 재고를 동기화하여 트래킹한다.
  • 전사 프로모션 및 매출 모니터링: 전사 단위 이벤트 프로모션을 등록/관리하며, 지점별 및 메뉴별 매출 통계를 분석하는 총괄 대시보드를 운영한다.

04. 사용자별 정밀 기능 정의서

기능 ID 행위 분류 요구 기능명 상세 요구 설명 명세 비즈니스 로직 및 연관 엔티티
U-001 소비자 주문 구성 (메뉴/사이즈) 메뉴, 맛, 컵/콘 및 사이즈를 선택하여 장바구니 세션을 구성 MENU, SIZE, CUP 동적 매핑 필터링 및 세션 바인딩
U-002 소비자 결제 수단 분기 최종 결제 시 현금 또는 신용카드 결제 방식을 선택하여 승인 PAYMENT 엔티티 생성. Pessimistic Lock 제어로 재고 원자적 차감 보장
U-003 소비자 포인트 적립 가상 키패드를 통한 휴대폰 번호 입력 및 결제액 비례 포인트 가산 MEMBER 조회. 신규 번호 시 임시 가입 처리 및 point Update
U-004 소비자 영수증 출력 결제 완료 트랜잭션 종료 후 프린터 SDK를 호출하여 내역 인쇄 ORDERS 데이터 영속성 획득 후 비동기 하드웨어 통신 호출
AS-001 지점 영업 개시/종료 통제 지점 포스에서 KIOSK 단말기의 구동 가능 상태를 원격 스위칭 KIOSK 테이블의 status 플래그 토글 및 클라이언트 락 해제
AS-002 지점 수동 발주 및 재고 관리 본사에 필요한 품목 발주 신청 및 지점 내 입고/폐기 상품 등록 처리 HQINVENTORY (요청 생성), BRANCHINVENTORY (가감 연산) 조작
AS-003 지점 주문 접수 및 취소 인입된 결제 건의 승인 처리 및 변심 시 대금 반환/재고 복구 ORDERS.status 변이 및 PAYMENT 취소 트랜잭션 롤백 보장
AS-004 지점 다차원 매출/판매량 통계 일일/월간 매출액 집계 및 판매 메뉴별(Flavor) 인기 순위 조회 SALES, ORDERITEMMENU 복합 인덱스 스캔 쿼리 및 렌더링
AH-001 본사 가맹점 마스터 관리 전국 신규 지점의 등록, 정보 갱신, 폐업/휴업 상태 삭제 제어 BRANCH 및 BRANCHMANAGER 엔티티 CRUD 라이프사이클 관리
AH-002 본사 메뉴/카테고리 배포 신제품(이달의 맛) 및 카테고리 분류 트리를 등록하고 전사 배포 CATEGORY, MENU, ITEM 마스터 데이터 변경 및 클라이언트 캐시 무효화
AH-003 본사 물류 승인 및 본사 재고 지점의 발주 요청을 검토 후 출하 승인하며 본사 물류창고 재고를 동기화 HQINVENTORY의 approval_status 및 delivery_status 파이프라인 제어
AH-004 본사 이벤트 프로모션 관리 전사 통합 할인, 신메뉴 비주얼 배너 등 프로모션 기간/조건 등록 PROMOTION 엔티티 및 COUPON 연계 규칙 엔진 반영
AH-005 본사 전사 통합 매출 통계 전국 가맹점의 통합 매출 누계 및 지점별/메뉴별 교차 분석 제공 Group By 및 Rollup을 활용한 SALES, ORDERS 대용량 DW 추출

05. 기능별 상세 세부 명세서

UML / ERD Database Binding Spec
U-001 ~ U-002 / 다단계 주문 객체 생성 및 결제 트랜잭션

소비자가 메뉴, 맛, 사이즈를 순차적으로 선택하면 프론트엔드 단에서 장바구니 세션을 유지합니다. 현금 또는 카드 결제 승인 시, Spring Boot 서버는 ORDERS 마스터 레코드를 Insert 하고, 외래키(FK)를 통해 ORDERITEM, ORDERITEMMENU 세부 내역 및 PAYMENT 승인 내역을 단일 데이터베이스 트랜잭션(@Transactional)으로 영속화하여 데이터 무결성을 보장합니다.

U-003 ~ U-004 / 멤버십 연동 포인트 적립 및 하드웨어 영수증

결제 단계 중 휴대폰 번호를 입력받아 MEMBER 테이블의 phone_number를 조회합니다. 존재할 경우 결제 금액의 일정 비율을 point에 누적 가산하며, 결제가 완전히 커밋된 후 프린터 SDK 비동기 큐를 호출하여 실물 영수증을 출력합니다.

AS-001, AS-003 / 키오스크 스위칭 및 주문(취소) 제어

점주는 POS 대시보드에서 KIOSK 엔티티의 영업 상태(Open/Close)를 스위칭하여 고객 단말의 주문 진입을 원격으로 차단/허용합니다. 잘못 인입된 주문은 ORDERS 상태를 Cancel로 업데이트하며, 차감되었던 BRANCHINVENTORY(로컬 재고)를 보상 트랜잭션을 통해 즉시 원상 복구(Rollback)합니다.

AS-002, AS-004 / 실재고 물류 발주 및 다차원 매출 시각화

매장 내 실물 재고 소진 및 폐기 발생 시 BRANCHINVENTORY를 수동 업데이트(입고/폐기 등록)하고, 부족 분에 대해 본사 테이블인 HQINVENTORY로 발주 신청 Insert 쿼리를 발생시킵니다. 또한 SALES 마스터를 조인하여 일별/월별 매출 차트를 렌더링하고, 인기 메뉴 통계를 도출합니다.

AH-001, AH-002, AH-004 / 마스터 데이터 및 이벤트 프로모션 통합 관리

BRANCH(가맹점), CATEGORY, MENU(상품 분류 및 메뉴) 등 전사 서비스 운영의 뼈대가 되는 핵심 마스터 데이터를 총괄 통제합니다. 신메뉴 출시 또는 PROMOTION(이벤트) 런칭 시 중앙 서버에서 데이터를 Insert/Update 하면 전국 지점 POS 및 Kiosk로 캐시가 무효화되어 동기화 배포됩니다.

AH-003, AH-005 / 글로벌 물류 승인 파이프라인 및 전사 매출 DW 분석

지점으로부터 상신된 발주 문건(HQINVENTORY)을 조회하여 재고 가용성을 판단한 뒤 승인/반려(approval_status) 처리를 수행하며 본사 창고 재고를 가감합니다. 전국의 누적된 SALES 데이터를 Data Warehouse로 집계하여 지점별 실적 비교, 시즌별 인기 메뉴 분석 등 경영진 의사결정용 대시보드를 제공합니다.

05. 홍보 통계 데이터 명세 및 기대 효과

01. 이달의 맛 홍보 데이터

속성: 이달의 맛 ID, 이름, 맛 설명, 대표 이미지(.png), 판매 시작/종료일, 메뉴판 노출 순서, 판매 여부

활용 방안: 새로 추가된 이달의 맛을 메뉴판 첫 번째에 고정 노출하여 신메뉴 홍보 및 구매를 적극적으로 유도합니다.

02. 인기 맛 통계 데이터

속성: 맛 ID, 이름, 누적 판매량, 판매 순위, 최근 한 달 판매량, 인기 메뉴 TOP1~TOP5 정보

활용 방안: 판매 데이터를 기반으로 인기 맛 순위를 산정하고, 메뉴판 옆에 TOP1~TOP5를 표시하여 고객의 메뉴 선택을 돕고 구매를 유도합니다.

03. 구매 시간대 통계 데이터

속성: 주문 ID, 주문 일시, 주문 시간대, 시간대별 주문 건수, 시간대별 매출액, 평균 주문 금액

활용 방안: 시간대별 구매 데이터를 분석하여 주문량이 적은 시간대를 파악하고, 해당 시간대에 할인 이벤트를 진행하여 매출을 증대시킵니다.

04. 할인 이벤트 데이터

속성: 이벤트 ID,명칭, 할인 적용 시작/종료 시간, 할인율, 적용 대상 메뉴, 이벤트 활성화 여부

활용 방안: 구매율이 낮은 시간대에 자동으로 할인 이벤트를 적용하여 유휴 타임슬롯의 오프라인 고객 방문을 능동적으로 유도합니다.

05. 스탬프 및 방문 쿠폰 데이터

속성: 회원 ID, 누적 주문 횟수, 쿠폰 ID, 할인 비율/금액, 발급/만료일시

활용 방안: 고객의 누적 방문 횟수에 따라 자동으로 쿠폰을 지급하여 지속적인 매장 방문과 단골 확보를 유도합니다. 특정 방문 횟수에 도달할 때마다 리워드를 제공함으로써 고객에게 목표 달성 성취감을 주고, 재방문 주기를 단축시키는 락인(Lock-in) 효과를 강화합니다.

EXPECTED EFFECTS | 전사 기대 효과
  1. 이달의 맛 우선 노출을 통한 신메뉴 홍보 강화

데이터베이스 도메인 영역별 테이블 역할 명세서 (rdb3.png 연동) Database Schema Catalog Map
영역 (도메인) 테이블명 (Table Name) 기능 및 역할 설명 명세
기준 정보
(Master Data)		 CATEGORY 상품 및 메뉴의 분류 체계 (카테고리) 관리
ITEM 입고 및 재고 관리의 기준이 되는 물리적 품목 정보
MENU 단품(맛) 단위의 실제 판매 메뉴 기본 정보
SIZE 판매 상품의 사이즈 규격, 가격 및 구성 정보
CUP 고객에게 제공되는 포장/취식 용기(컵, 콘 등) 규격 정보
MENUOPTION 메뉴에 추가 가능한 선택형 옵션(토핑 등) 부가 정보
매장 운영
(Store & Operation)		 HQMANAGER 본사 시스템을 총괄 및 제어하는 관리자 계정 정보
BRANCHMANAGER 각 가맹점을 책임지는 지점 관리자(점주) 계정 정보
BRANCH 전국 가맹 지점의 식별, 위치 및 운영 기본 정보
KIOSK 매장 내 설치된 개별 키오스크 기기의 상태 및 매핑 관리
SALES 지점별 일일 결제 정산 및 누적 매출 통계 데이터 보관
주문 및 결제
(Order & Transaction)		 CUSTOMER 결제를 진행하는 회원 및 비회원 공통 기본 고객 정보
MEMBER 포인트 및 정보가 보관된 가입 회원의 상세 정보
ORDERS 개별 결제 단위로 묶인 통합 주문 마스터(영수증 단위) 트랜잭션
ORDERITEM 하나의 주문 번호에 속한 단위 사이즈별 상품 주문 내역
ORDERITEMMENU 해당 상품(사이즈) 내에서 고객이 선택한 세부 맛(Flavor) 구성 내역
ORDERITEMOPTION 주문 상품에 추가로 부착된 토핑 및 부가 옵션 선택 내역
PAYMENT PG 승인 내역, 결제 금액, 결제 시간 및 결제 수단(카드/현금 등) 기록
재고 및 발주
(Inventory & Supply)		 BRANCHINVENTORY 각 지점별로 보유하고 있는 품목의 실시간 재고량 추적
HQINVENTORY 지점의 물류 발주 요청 및 본사와의 물류 승인/배송 등 이동 내역 관리
마케팅 및 홍보
(Marketing)		 PROMOTION 이벤트 및 프로모션 행사의 기획 정보 및 기간 설정 제어
COUPON 회원에게 개별 발급된 할인 쿠폰 고유 식별 및 사용 만료 여부 추적
고객지원 및 소통
(CS / Board)		 CS 지점 관리자와 본사 총괄 관리자 간의 문의사항 피드백 및 소통 게시판
MySQL 8.0 Physical DDL Syntax Standard 

create database sweetscoop default character set utf8; 
use sweetscoop;

create table branch
(
  id          int          not null auto_increment comment '지점 id',
  branch_name varchar(50)  not null comment '지점명',
  location    varchar(100) null     comment '위치/주소',
  primary key (id)
) comment '지점 정보';

create table branchinventory
(
  branch_id   int not null comment '지점 id',
  item_id     int not null comment '물품 id',
  stock_level int null     default 0 comment '현재 재고량 (단위: g 등)'
) comment '지점별 실시간 재고';

create table branchmanager
(
  id        varchar(50) not null comment '점주 id',
  branch_id int         not null comment '담당 지점 id',
  primary key (id)
) comment '지점 관리자/점주';

create table category
(
  id   int         not null comment '카테고리 id',
  name varchar(30) not null comment '카테고리명(아이스크림,커피,아이스모찌 등)',
  primary key (id)
) comment '카테고리';

create table coupon
(
  id             int      not null auto_increment comment '쿠폰 id',
  member_id      int      not null comment '회원 id',
  issue_date     datetime null     comment '발급일시',
  expiry_date    datetime null     comment '만료일시',
  discount_value double   null     comment '할인 비율/금액',
  is_used        boolean  null     default false comment '사용 여부',
  used_at        datetime null     comment '사용 완료 일시',
  primary key (id)
) comment '발급 쿠폰 이력';

create table cs
(
  id           int         not null auto_increment comment '문의 id',
  title        varchar(50) not null comment '문의 제목',
  content      text        null     comment '문의 내용',
  created_at   datetime    null     comment '문의일시',
  manager_id   varchar(50) not null comment '작성 점주 id',
  hqmanager_id varchar(50) null     comment '담당 본사 관리자 id',
  primary key (id)
) comment '문의게시판';

create table cup
(
  id               int         not null comment '컵 id',
  name             varchar(30) not null comment '형태 명칭(컵,콘,와플 등)',
  additional_price int         not null default 0 comment '컵 선택 시 추가 금액',
  primary key (id)
) comment '제공 타입(컵/콘)';

create table customer
(
  id            int         not null auto_increment comment '고객 고유 id',
  customer_type varchar(50) null     comment '비회원/회원 구분 플래그',
  primary key (id)
) comment '고객 일반 (회원/비회원 공통)';

create table hqinventory
(
  id               int         not null auto_increment comment '발주 요청 id',
  branch_id        int         not null comment '요청 지점 id',
  item_id          int         not null comment '물품 id',
  hqmanager_id     varchar(50) null     comment '승인 담당 관리자 id',
  approval_status  varchar(20) null     comment '승인 상태',
  delivery_status  varchar(20) null     comment '배송 상태',
  request_quantity int         not null comment '요청 수량',
  primary key (id)
) comment '본사-지점 간 발주/물류 이력';

create table hqmanager
(
  id   varchar(50) not null comment '관리자 id',
  name varchar(50) not null comment '관리자 이름',
  primary key (id)
) comment '본사 관리자 (기본 정보)';

create table item
(
  id          int         not null auto_increment comment '물품 id',
  category_id int         not null comment '카테고리 id',
  unit        int         not null comment '통/박스 단위(입고 기준)',
  item_name   varchar(50) not null comment '물류용 실물품명',
  primary key (id)
) comment '실제 물류/재고 원자재';

create table kiosk
(
  id        int         not null auto_increment comment '기기 고유 id',
  branch_id int         not null comment '소속 지점 id',
  status    varchar(20) not null comment '기기 상태 (정상, 고장 등)',
  primary key (id)
) comment '개별 키오스크 기기';

create table member
(
  id           int         not null auto_increment comment '회원 고유 id',
  customer_id  int         not null comment '고객 id',
  phone_number varchar(20) null     comment '연락처',
  order_count  int         null     default 0 comment '누적 주문 횟수',
  point        int         null     default 0 comment '적립금',
  created_at   datetime    null     comment '회원 생성일시',
  primary key (id)
) comment '회원 상세 정보';

create table menu
(
  id          int         not null comment '메뉴 id',
  category_id int         not null comment '카테고리 id',
  name        varchar(30) null     comment '메뉴/맛 이름(엄마는 외계인 등)',
  item_id     int         not null comment '물품 id',
  primary key (id)
) comment '실제 판매용 메뉴';

create table menu_option
(
  id          int         not null comment '옵션 id',
  category_id int         not null comment '카테고리 id',
  name        varchar(30) not null comment '옵션 명칭',
  price       int         not null comment '옵션 추가 가격',
  is_active   boolean     not null default true,
  primary key (id)
) comment '선택형 옵션(hot, ice 등)';

create table orderitem
(
  id          int not null auto_increment comment '상세 주문 id',
  order_id    int not null comment '소속 주문 번호',
  cup_id      int not null comment '선택 컵 id',
  size_id     int not null comment '선택 사이즈 id',
  quantity    int not null default 1 comment '주문 수량',
  total_price int not null comment '옵션/컵 포함 최종 개별 금액',
  primary key (id)
) comment '주문 상품(1개 사이즈 단위), 주문에 들어있는 실제 메뉴';

create table orderitemmenu
(
  id            int not null auto_increment comment '선택 맛 상세 id',
  order_item_id int not null comment '상세 주문 id',
  menu_id       int not null comment '메뉴(맛) id',
  primary key (id)
) comment '주문 상품의 세부 맛 선택 내역';

create table orderitemoption
(
  id             int not null auto_increment comment '선택 옵션 상세 id',
  order_item_id  int not null comment '상세 주문 id',
  menu_option_id int not null comment '추가 옵션 id',
  primary key (id)
) comment '주문 상품의 추가 토핑/옵션 내역';

create table orders
(
  id          int         not null auto_increment comment '주문 번호',
  customer_id int         not null comment '고객 id (비회원 포함)',
  branch_id   int         not null comment '지점 id',
  kiosk_id    int         not null comment '주문 발생 기기 id',
  order_type  varchar(20) not null comment '유형 (매장/포장 등)',
  language    varchar(20) not null comment '언어 설정',
  status      varchar(20) not null comment '주문 상태(결제완료/준비중/완료)',
  created_at  datetime    not null comment '주문 발생 시각',
  waiting_no  int         null     comment '고객 호출 번호',
  receipt_no  varchar(50) not null comment '영수증 발급 번호',
  total_price int         not null comment '총 주문 금액',
  primary key (id)
) comment '통합 주문 내역';

create table payment
(
  id                int          not null auto_increment comment '결제 id',
  order_id          int          not null comment '주문 번호',
  method            varchar(20)  not null comment '결제 수단',
  amount            int          not null comment '결제 총 금액',
  card_company      varchar(30)  not null comment '결제 카드사',
  payment_time      datetime     not null comment '결제 시간',
  payment_status    varchar(30)  null     comment '상태(승인/취소/실패)',
  pg_transaction_id varchar(100) null     comment 'pg사 결제 고유 번호',
  point_used        int          null     default 0 comment '사용 포인트',
  point_earned      int          null     default 0 comment '적립 포인트',
  primary key (id)
) comment '결제 승인 내역';

create table promotion
(
  id         int          not null auto_increment comment '이벤트 id',
  event_name varchar(100) not null comment '이벤트명',
  start_date date         null     comment '시작일',
  end_date   date         null     comment '종료일',
  primary key (id)
) comment '이벤트/홍보';

create table sales
(
  id           int  not null auto_increment comment '매출 id',
  branch_id    int  not null comment '지점 id',
  sales_date   date not null comment '매출 일자 (일일 정산용)',
  total_amount int  null     default 0 comment '총 매출액',
  primary key (id)
) comment '일별 지점 매출 통계';

create table size
(
  id             int         not null auto_increment comment '사이즈 id',
  category_id    int         not null comment '카테고리 id',
  name           varchar(50) not null comment '사이즈명(싱글,파인트 등)',
  flavor_cnt     int         not null comment '선택 가능 맛 갯수',
  price          int         not null comment '사이즈 가격',
  total_weight_g int         not null comment '해당 사이즈 총 제공 중량(g) - 재고 차감 기준',
  primary key (id)
) comment '상품 사이즈 규격';
